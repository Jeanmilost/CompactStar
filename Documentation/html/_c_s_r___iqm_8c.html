◆ CSR_EIQMVertexArrayType

enum CSR_EIQMVertexArrayType

Vertex array type

Note
All vertex array entries must ordered as defined below, if present. I.e. position comes before normal, comes before ..., comes before custom, where a format and size is given, this means models intended for portable use should use these an IQM implementation is not required to honor any other format/size than those recommended. However, it may support other format/size combinations for these types if it desires

Creates a IQM model from a buffer

Parameters
pBuffer- buffer containing the IQM data to read
pVertFormat- model vertex format, if 0 the default format will be used
pVertCulling- model vertex culling, if 0 the default culling will be used
pMaterial- mesh material, if 0 the default material will be used
meshOnly- if 1, only the mesh will be drawn. All other data will be ignored
poseOnly- if 1, the model will take the default pose but will not be animated
fOnGetVertexColor- get vertex color callback function to use, 0 if not used
fOnLoadTexture- called when a texture should be loaded
fOnApplySkin- called when a skin should be applied to the model
fOnDeleteTexture- callback function to notify the GPU that a texture should be deleted
Returns
the newly created IQM model, 0 on error
Note
The IQM model must be released when no longer used, see csrIQMModelRelease()
